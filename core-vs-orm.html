What is SQLAlchemy?

SQLAlchemy is a Python database toolkit created by Mike Bayer in 2005. It's the most widely used database library in the Python ecosystem — and it's much more than just an ORM.

At its core, SQLAlchemy gives you two things: a powerful SQL Expression Language for writing SQL as Python, and a full Object Relational Mapper for working with databases through Python classes. You can use either layer — or both together.

Unlike other ORMs that hide SQL from you, SQLAlchemy keeps you in control. It generates efficient SQL, but never stops you from going lower-level when you need to.

Not Just an ORM

Django has a built-in ORM. Peewee is a simple ORM. SQLAlchemy is a complete database toolkit — the ORM is just one part of what it offers.

Core vs ORM — Two Layers, One Library

SQLAlchemy has two distinct layers that work together. Understanding the difference helps you choose the right tool for each situation — and most production apps use both.

SQLAlchemy Core
The SQL Expression Language layer
SQL Expression Language
Build SQL as Python objects — select(users).where(...)
Table & MetaData objects
Define schema programmatically — no classes required
Connection-based execution
Explicit conn.execute() — you control every step
Best for…
Data pipelines, bulk operations, reporting, fine-grained SQL control
SQLAlchemy ORM
The Object-Relational Mapping layer
Python Class Models
Define a class = define a table. Pure Python, no SQL strings
Session-based workflow
Track changes to objects, commit once — SQLAlchemy handles the rest
Relationship mapping
Navigate user.orders like Python lists — JOINs handled automatically
Best for…
Web apps, APIs, CRUD-heavy applications, rapid development

Same Task — Two Approaches

USING CORE 
from sqlalchemy import (
    create_engine, Table, Column,
    Integer, String, MetaData, select
)

metadata = MetaData()
users = Table('users', metadata,
    Column('id', Integer, primary_key=True),
    Column('name', String(100)),
    Column('age', Integer),
)

with engine.connect() as conn:
    stmt = select(users).where(
        users.c.age > 18
    )
    result = conn.execute(stmt)
    for row in result:
        print(row.name)
USING ORM 
from sqlalchemy.orm import (
    DeclarativeBase, Session,
    Mapped, mapped_column
)
from sqlalchemy import Integer, String

class User(Base):
    __tablename__ = 'users'
    id:   Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str]
    age:  Mapped[int]

with Session(engine) as session:
    users = session.query(User).filter(
        User.age > 18
    ).all()

    for user in users:
        print(user.name)
Which Should You Use?

For most web apps and APIs, start with the ORM. It's faster to write and easier to maintain. Drop down to Core when you need complex SQL, bulk inserts, or performance-critical queries where you want explicit control.
