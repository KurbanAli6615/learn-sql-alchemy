Installation & Basic Configuration

You need two things: SQLAlchemy (the toolkit) and a database driver (the connector). The driver depends on which database you're using and whether your app is sync or async.

For learning, start with SQLite — no installation, no server, just a file. Switch to PostgreSQL when you're ready for production.

sqlalchemy
The main toolkit — ORM, Core, engine, session. The only required package.
asyncpg
Async PostgreSQL driver. Needed for async apps using FastAPI or asyncio.
psycopg2-binary
Sync PostgreSQL driver. Use for sync apps or Django-style projects.
alembic
Database migration tool. Handles schema changes safely over time. (Module 15)
SQLite needs no driver

Python includes a SQLite driver built-in. create_engine("sqlite:///app.db") works out of the box — great for learning and testing.

TERMINAL
# Async PostgreSQL (FastAPI, modern apps)
pip install sqlalchemy[asyncio] asyncpg

# Sync PostgreSQL
pip install sqlalchemy psycopg2-binary

# SQLite only (no extra driver)
pip install sqlalchemy

# Full production setup
pip install sqlalchemy[asyncio] asyncpg alembic
FIRST WORKING EXAMPLE — SQLITE
from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase, Session
from sqlalchemy.orm import Mapped, mapped_column

engine = create_engine("sqlite:///myapp.db")

class Base(DeclarativeBase): pass

class User(Base):
    __tablename__ = "users"
    id:   Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str]

Base.metadata.create_all(engine)  # create the table

with Session(engine) as session:
    session.add(User(name="Alice"))
    session.commit()
    print("User saved!")  # ✓

create_async_engine — What It Does

The engine is the heart of SQLAlchemy. It manages how your app connects to the database. You create it once when your app starts, and use it for the entire lifetime of the application.

You give it a database URL and some settings, and it builds a Connection Pool — a pre-warmed set of connections ready for immediate use. No new connection is opened for every request.

create_async_engine is the async version, needed for FastAPI and any app using asyncio.

database.py
from sqlalchemy.ext.asyncio import (
    create_async_engine,
    async_sessionmaker
)

DATABASE_URL = (
    "postgresql+asyncpg://"
    "user:password@localhost:5432/mydb"
)

engine = create_async_engine(
    DATABASE_URL,
    pool_pre_ping=True,
    pool_recycle=3600,
    pool_size=10,
    max_overflow=20,
)

# session factory — use this to open sessions
async_session = async_sessionmaker(
    engine, expire_on_commit=False
)
Create Once, Use Everywhere

Create the engine once at startup — never inside a function that runs per request. Creating an engine is expensive; using it is fast.

Connection Pool — The 4 Settings Explained

The connection pool is like a waiting room of database connections — always ready to serve, so your app doesn't need to create a brand-new connection for every single request. These 4 settings control how that pool behaves.

The Session Lifecycle — yield vs return

Every beginner asks this: "Why does db_session use yield instead of return?" The answer is simple — yield lets the function pause, give out the session, and then wake back up to do cleanup when your code is done. return ends the function immediately — no cleanup ever happens.

PROBLEM
Using return — session leaks
DON'T DO THIS
def get_session():
    session = Session(engine)
    return session  # function ENDS here
    # cleanup code here? NEVER runs!

# caller:
session = get_session()
user = session.query(User).first()
# ❌ nobody commits the transaction
# ❌ nobody closes the session
# ❌ connection stays open forever
# ❌ pool runs out → app freezes
CORRECT
Using yield — automatic cleanup
DO THIS ✓
async def db_session():
    async with async_session() as session:
        async with session.begin():
            yield session  # ⏸ PAUSE — caller runs
            # ▶ RESUME — caller is done
        # ✓ auto-commit OR auto-rollback
    # ✓ session closed automatically
    # ✓ connection returns to pool

Steps 1–3: Setup
Before your code touches the database, SQLAlchemy does the preparation — grabs a connection from the pool, wraps it in a session object, and starts a transaction. A transaction is like Google Docs "draft mode" — all changes are held temporarily, not written to the real database yet.
Step 4: The Pause (yield)
This is the magic. yield freezes db_session() right at this line. Python hands the session to FastAPI's dependency injection, which passes it to your route as the session parameter. The function is still "alive" — it's just waiting.
Step 5: Your Code Runs
This is where your route function executes. Every session.query(), session.add(), or session.execute() call runs here. None of it hits the database permanently yet — it's all in the open transaction (the draft).
Step 6–7: Cleanup
When your route returns a response, FastAPI signals that the dependency is done. db_session() wakes up.

If your route succeeded: the async with session.begin() block auto-commits — the draft becomes permanent.

If an exception was raised: it auto-rolls back — the draft is discarded, the database is unchanged. Then the session closes and the connection goes back to the pool.

You never write a single line of cleanup code.
How to Wire It in FastAPI
YOUR ROUTE
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from database import db_session

router = APIRouter()

@router.get("/users/{user_id}")
async def get_user(
    user_id: int,
    # FastAPI calls db_session() for you
    session: AsyncSession = Depends(db_session)
):
    user = await session.get(User, user_id)
    if not user:
        raise HTTPException(404)
    return user  # ← session auto-closes here
What Depends(db_session) does automatically:
1
Calls db_session(), runs it up to the yield
2
Injects the yielded session as your route's parameter
3
Your route runs — queries, inserts, whatever
4
FastAPI resumes db_session() past the yield — commit, close, pool
You write zero cleanup code. The yield + async with pattern handles it all.

1
Create the engine and session database.py
database.py
from typing import AsyncIterator
from sqlalchemy.ext.asyncio import (
    AsyncSession, async_sessionmaker, create_async_engine
)
from sqlalchemy.orm import DeclarativeBase

# SQLite async — great for learning, no server needed
DATABASE_URL = "sqlite+aiosqlite:///./myapp.db"

engine = create_async_engine(DATABASE_URL, echo=True)

# Session factory — use this everywhere
async_session = async_sessionmaker(engine, expire_on_commit=False)

# Base class — all models inherit from this
class Base(DeclarativeBase): pass

# The session dependency for FastAPI
async def db_session() -> AsyncIterator[AsyncSession]:
    async with async_session() as session:
        async with session.begin():
            yield session
2
Define your models (tables) models.py
models.py
from sqlalchemy.orm import Mapped, mapped_column
from sqlalchemy import String
from database import Base

class User(Base):
    __tablename__ = "users"

    id:    Mapped[int] = mapped_column(primary_key=True)
    name:  Mapped[str] = mapped_column(String(100))
    email: Mapped[str] = mapped_column(String(200), unique=True)

    def __repr__(self):
        return f"User(id={self.id}, name={self.name!r})"
3
Build the FastAPI routes main.py
main.py
from contextlib import asynccontextmanager
from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from database import engine, Base, db_session
from models import User

# Create tables when the app starts
@asynccontextmanager
async def lifespan(app: FastAPI):
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    yield

app = FastAPI(lifespan=lifespan)

# POST /users — create a user
@app.post("/users")
async def create_user(
    name: str, email: str,
    session: AsyncSession = Depends(db_session)
):
    user = User(name=name, email=email)
    session.add(user)
    await session.flush()  # gets the auto-generated id
    return {"id": user.id, "name": user.name}

# GET /users — list all users
@app.get("/users")
async def list_users(
    session: AsyncSession = Depends(db_session)
):
    result = await session.execute(select(User))
    return result.scalars().all()
